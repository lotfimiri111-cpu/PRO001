# 🔐 إعداد نظام الدفع — مذكرتي Pro v20 Ultra

## ما الجديد في v18؟

### نظام الدفع الكامل:
- ✅ **Preview آمن** للشرائح بـ Watermark (بدون تحميل)
- ✅ **نظام الطلبات**: إنشاء طلب + رفع وصل الدفع
- ✅ **لوحة Admin** للمراجعة والقبول/الرفض
- ✅ **أكواد التفعيل** لمرة واحدة مع صلاحية 48 ساعة
- ✅ **سجل Audit** لكل عملية (وقت، IP، فاعل)
- ✅ **ملفات PPTX محمية** على الخادم لا تُرسل إلا بعد الكود

---

## إعداد كلمة مرور Admin

```bash
# توليد hash لكلمة المرور (مثال: "mysecretpass")
python3 -c "import hashlib; print(hashlib.sha256(b'mysecretpass').hexdigest())"
```

ثم ضع الناتج في متغير البيئة على Render:
```
ADMIN_PASSWORD_HASH=<الناتج>
```

---

## متغيرات البيئة المطلوبة على Render

| المتغير | الوصف | مثال |
|---------|-------|------|
| `ADMIN_PASSWORD_HASH` | SHA256 لكلمة مرور Admin | `sha256(...)` |
| `SECRET_KEY` | مفتاح Flask (عشوائي) | `render: generateValue: true` |
| `STORAGE_DIR` | مسار تخزين الملفات | `/opt/render/project/src/storage` |
| `DB_PATH` | مسار قاعدة البيانات | `/opt/render/project/src/mathkarati_payments.db` |

> ⚠️ **تنبيه Render**: على الخطة المجانية، يتم مسح الملفات عند إعادة النشر.
> استخدم **Render Disk** أو **خدمة تخزين خارجية** للإنتاج.

## مسار العمل الكامل

```
1. الطالب يملأ البيانات → ينشئ العرض
2. يظهر Preview مقفل بـ Watermark (تصفح الشرائح بدون تحميل)
3. الطالب ينقر "اشتري الآن" → يختار CCP أو BaridiMob
4. يرى معلومات الحوالة → يدفع → يرفع الوصل
5. يصل الطلب للوحة Admin (/admin)
6. Admin يفتح الطلب → يشاهد الوصل → يضغط "قبول"
7. يُولَّد كود 8 أحرف تلقائياً (صالح 48 ساعة)
8. Admin ينسخ الكود → يرسله للطالب (واتساب / إيميل)
9. الطالب يدخل الكود → يتحمل PPTX الكامل
10. الكود يُلغى تلقائياً بعد الاستخدام
```

## لوحة Admin — `/admin`

- **المرور الافتراضي**: `admin1234` (غيّره فوراً!)
- **الوصول**: `https://your-app.onrender.com/admin`
- **الميزات**:
  - إحصائيات الإيرادات والطلبات
  - مشاهدة وصل الدفع (صورة أو PDF)
  - قبول/رفض الطلبات مع ملاحظة
  - توليد وإعادة توليد أكواد التفعيل
  - سجل أحداث كامل لكل طلب
